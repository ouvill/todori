# ADR-001: E2EEを製品の核として厳格適用する

**日付**: 2026-07-03
**状態**: 採用

### 決定

サーバーはDEKで暗号化されたblobと最小限の同期メタデータのみを保持する。サーバー側でコンテンツの復号・解釈・マージは一切行わない。

### 理由

E2EEはこの製品の譲れない価値定義そのものである（Proton等の思想への共感）。

### 帰結

- サーバー側マージ不可のため、クライアント側マージが必須となる。
- 検索・ソートのため平文のローカルDB（SQLCipher保護）が必須となる。
- サーバー起点プッシュ通知は不可であり、リマインダーはローカル通知で実現する。

### 却下した代替案

該当なし（本ADRは前提となる原則の確認であり、対案の比較検討を伴わない）。
