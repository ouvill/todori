# 設計判断記録（ADR）

E2EE Todoアプリ「Taskveil」

---

設計上の重要な意思決定と、その判断理由を記録する。仕様の詳細は [`03_技術仕様書.md`](./03_技術仕様書.md) を正とし、ADRは「なぜそう決めたか」を保存する。

ADR本文の正本は [`docs/adr/`](./adr/) の個別ファイルである。このファイルは既存のパスと見出しanchorを維持する互換索引として扱う。

## 運用規約

- ADRは `docs/adr/ADR-NNN.md` の1件1ファイルで管理する。番号は3桁の連番とし、採番済みIDを再利用しない。
- 作業branchで追加するADR番号は暫定とする。merge前に対象branchの最新状態を取り込み、同じ番号が先に採番済みなら個別ファイル、見出し、索引、参照を次の空き番号へ変更する。
- 各ADRは日付と状態を持つ。状態は原則として `提案`、`採用`、`非推奨`、`置換済み` のいずれかを用いる。
- 採用済みADRの判断を後から書き換えず、判断を変更する場合は新しいADRを追加して旧ADRを `置換済み` とする。
- ADRを追加または状態変更したときは、個別ファイルとこの索引を同じ変更で更新する。
- 文書やコードから新しく参照するときは個別ADRへ直接リンクする。既存の `docs/05_設計判断記録.md ADR-NNN` 参照は互換索引で案内を維持する。
