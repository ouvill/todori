## 1. UIモード

### F-01 初回起動時のUIモード選択

初回起動時、ユーザーは「シンプルUI」と「高機能UI」のいずれかを選択する。選択は設定画面からいつでも切替可能とする。切替時にデータの損失・変換は発生しない。

### F-02 シンプルUI

リスト・タスク・完了チェックのみで構成する最小構成のUIとする。カレンダーやかんばんなど高度な機能へのアクセスは提供しない。初めてタスク管理アプリに触れるユーザーや、シンプルな運用のみを望むユーザーを対象とする。

### F-03 高機能UI

タブまたはサイドバーから以下の各画面にアクセスできる。

- Today（F-12）
- カレンダー（F-13）
- かんばん（F-14）
- 統計（F-22）
- タイマー（F-16, F-17）

### F-04 デザイントーン

角丸を基調とした柔らかく・親しみやすく・エレガントな見た目とし、**エメラルドをブランドカラー**とする淡色パレットを採用する。ライトテーマ・ダークテーマの両方を提供し、OS設定に追従する自動切替と手動切替の両方をサポートする。

## 2. タスク管理

### F-05 タスクのCRUD

タスクは作成・参照・更新・削除が可能である。タスクは以下のフィールドを持つ。

| フィールド | 説明 |
|---|---|
| タイトル | 必須。1行テキスト |
| ノート | 任意。Markdown形式の詳細メモ |
| ステータス | `todo` / `in_progress` / `done` / `wont_do`（F-06参照） |
| 優先度 | なし / 低 / 中 / 高 |
| 締切日時（due） | 任意。日時指定 |
| 作業予定日時（scheduled） | 任意。日時指定 |
| 見積所要時間 | 任意。分単位 |
| 通知時刻 | 任意。複数設定可（F-25参照） |
| タグ | 任意。複数付与可（F-10参照） |
| 所属リスト | 必須。1タスク1リスト（F-09参照） |
| 親タスク | 任意。サブタスクの場合に設定（F-06参照） |
| 並び順 | リスト内での表示順序を保持する内部値 |

### F-06 ステータス遷移とサブタスク

ステータスは `todo` → `in_progress`（任意経由） → `done` / `wont_do` の順に遷移する。`in_progress` を経由せず `todo` から直接 `done` / `wont_do` へ遷移することも可能である。

`done` および `wont_do` はタスクを削除せず保存し、いつでも一覧・検索から閲覧および再オープン（`todo` へ戻す）が可能である。`wont_do` は GitHub Issue の「Close as not planned」に相当し、対応しないと判断したタスクを完了扱いにはせず区別して記録する用途に用いる。

サブタスクは無制限に入れ子にできる。親タスクを完了操作した際、配下に未完了のサブタスクが存在する場合は確認ダイアログを表示し、ユーザーの意思確認を行う。親タスクには配下サブタスクの完了率（進捗率）を表示する。

### F-07 削除・Undo（2026-07-07改訂: docs/05 ADR-009参照）

タスクの削除は恒久削除とする。ゴミ箱は設けない。削除操作は一覧からの即時操作ではなく、タスク詳細画面のサブメニューから起動し、「元に戻せません／永久に失われます」という追加確認を必須とする。

Undoは完了（チェック）操作および編集操作を対象とする。削除はUndo対象外とする（起動導線の深さおよび不可逆警告の追加確認による二段防御で誤削除を防止する）。

リストを削除する場合、配下のタスク（完了済みを含む）も恒久削除される。削除確認ダイアログでは影響を受けるタスク件数（完了済みを含む）を明示し、履歴を残したい場合はリストのアーカイブ（F-09参照）を利用するよう誘導する。

### F-08 並び替え

タスク一覧は以下の並び替え方式をサポートする。

- 手動並び替え（ドラッグ&ドロップ）
- 締切順
- 優先度順
- 作成順

## 3. リスト・タグ

### F-09 リスト

ユーザーは複数のリストを作成できる。リストは名前・色・アイコンを設定可能とする。初期状態では「インボックス」リストがデフォルトとして用意され、リスト未指定のタスクの受け皿となる。

リストはアーカイブ・アーカイブ解除が可能である（2026-07-07改訂: docs/05 ADR-009参照）。アーカイブされたリストは通常のリスト一覧から分離して表示する（別セクションまたは専用画面）。アーカイブ時、リストのデータおよび完了済みタスクを含む履歴は完全に保全される。

### F-10 タグ

タグはリストを横断して複数のタスクに付与可能とする。タグ・リスト・ステータス・期日を組み合わせたフィルタ表示をサポートする。

### F-11 全文検索

タスクのタイトル・ノートを対象としたローカル全文検索を提供する。検索インデックスは端末内にのみ保持し、E2EE設計のためサーバー側での検索（サーバーサイド全文検索）は行わない。

## 4. ビュー

### F-12 Todayビュー

日付のみ期限（その日中）または日時指定期限（その時刻まで）が本日であるタスクに加え、作業予定日時（scheduled）が本日のタスクと期限超過タスクを集約して表示する。日付のみ期限は翌ローカル日から、日時指定期限は指定instant以降に期限超過とする（2026-07-12改訂: ADR-017）。

### F-13 カレンダー表示

月表示・週表示を切り替え可能なカレンダー画面を提供する。日付のみ期限、日時指定期限、作業予定日時をそれぞれの意味を保ってカレンダー上にプロットし、タスクをドラッグすることで日付変更が可能である。

### F-14 かんばん表示（高機能UIのみ）

ステータス列（`todo` / `in_progress` / `done` / `wont_do`）またはリスト列を軸としたかんばんボードを提供する。ドラッグ&ドロップで列間の移動（ステータス変更またはリスト変更）が可能である。

### F-15 完了ログビュー

過去に `done` または `wont_do` となったタスクを、日付・リスト・タグで絞り込んで振り返るための一覧画面を提供する。

## 5. タイマー

### F-16 Pomodoroタイマー

作業時間25分・休憩時間5分を初期値とし、それぞれカスタム可能なPomodoroタイマーを提供する。長休憩に入るまでのセッション間隔（例: 4セッションごと）も設定可能とする。各セッションはタスクに紐付けて記録し、そのタスクの作業実績として蓄積する。

### F-17 通常作業タイマー（ストップウォッチ型）

Pomodoro形式によらない、開始・停止のみのストップウォッチ型タイマーを提供する。計測した実作業時間はタスクに記録する。

### F-18 見積 vs 実績比較

タスクに設定した見積所要時間と、タイマー（F-16, F-17）で記録された実績時間を比較し、統計画面（F-22）に反映する。

## 6. テンプレート・繰り返し

### F-19 テンプレート

サブタスク構造と内容フィールドをテンプレートとして保存できる。v1 snapshotはtitle、note、priority、estimated minutes、親子構造、順序だけを含み、status、日時、reminder、timer、assignee、ID、rank、tagは含めない。tag対応後はsnapshot schema revisionを上げて追加する。保存したテンプレートはワンタップでタスクとして起票（インスタンス化）できる（2026-07-17改訂: ADR-021）。

### F-20 繰り返しタスクの自動登録

テンプレート（F-19）に繰り返しルールを組み合わせることで、Todoを自動生成する。繰り返しルールは RRULE 準拠とし、毎日 / 毎週特定曜日 / 毎月特定日 / カスタムルールをサポートする。

生成処理はローカル端末上で実行する。オフライン状態が続いた場合でも、次回アプリ起動時に未生成分を遡って補完生成する。

### F-21 習慣トラッキング（streak）【追加提案機能】

繰り返しタスク（F-20）の連続する予定回の達成状況を記録し、streak（連続達成回数）として表示する。各予定回は次の予定回より前にdoneで達成とし、有限規則の最終回は終了条件を除いた同じcadenceのvirtual next occurrenceを期限とする（2026-07-17改訂: ADR-021）。

---

## 7. 統計・レビュー【追加提案機能】

### F-22 統計ダッシュボード

以下の指標を可視化する統計画面を提供する。

- 完了数の推移（日次/週次/月次）
- Pomodoro実績（セッション数・合計時間）
- 見積精度（見積所要時間と実績時間の乖離）
- タグ別・リスト別の時間配分

### F-23 週次レビュー画面

週単位で以下を振り返る画面を提供する。

- 今週完了したタスク
- 持ち越したタスク（未完了のまま翌週へ繰り越されたもの）
- 今週 `wont_do` としたタスク

## 8. 通知

### F-24 ローカル通知

通知はすべて端末内で完結するローカル通知として実装する。E2EE設計上、サーバーはタスクの内容だけでなく通知時刻そのものも把握できないため、通知はサーバー経由ではなく端末のローカルスケジューリング機能のみで実現する。

### F-25 複数通知時刻・スヌーズ

1つのタスクに対して複数の通知時刻を設定できる。通知に対するスヌーズ（後で再通知）操作をサポートする。

### F-26 デスクトップ通知

Windows / macOS / Linux 各OSのネイティブ通知機能を用いて通知を表示する。

## 9. アカウント・同期（Pro）

### F-27 ローカル利用

アカウント登録なしでローカル完結で全機能（同期を除く）を利用できる。

### F-28 E2EE同期（Pro課金）

アカウント登録することでE2EE同期機能が有効化される。同期機能はPro課金プランの対象とする。

### F-29 デバイス管理

登録済みアカウントに対して複数デバイスをリンクできる。デバイス一覧画面から、リンク済みデバイスの確認および個別の失効（同期対象からの除外）が可能である。

### F-30 オフライン編集と自動同期

オフライン中に行われた編集内容は端末内に保持され、オンライン復帰時に自動的にサーバーへ同期される。複数デバイスでの編集が競合した場合は自動マージを行う。マージアルゴリズムの詳細は技術仕様書を参照する。

### F-31 リカバリーキー

アカウント登録時にリカバリーキーを発行し、ユーザーに表示および保存を強く促す。パスワードを忘れた場合、リカバリーキーによってのみアカウントおよびデータへの復旧が可能である。パスワードとリカバリーキーの両方を喪失した場合、E2EE設計上いかなる手段でもデータ復旧は不可能であることをUI上に明示する。

## 10. Organization（Org課金）

### F-32 Org作成・メンバー管理

Organizationを作成し、招待リンクと鍵交換によりメンバーを招待できる。メンバーには Owner / Admin / Member のロールを設定する。

### F-33 Org内共有リスト

Organization内でリストを共有し、タスクを複数メンバー間で共有できる。

### F-34 担当者アサイン・コメント【追加提案機能】

共有タスクに対する担当者のアサイン、およびタスクへのコメント機能を提供する。

### F-35 監査ログ【追加提案機能】

Organization内での操作（誰が・いつ・何を変更したか）を記録する監査ログを提供する。タスク内容自体は暗号化されたままであり、ログに記録されるのは操作メタデータ（操作者・操作日時・操作種別等）のみとする。

### F-54 ワークスペース切替

個人スペースと参加している各Organizationスペースは、別ワークスペースとしてUI上分離され、画面上部または側面のスイッチャーで切り替える。タグ・テンプレート・スケジュール・タイマー記録等はワークスペースごとに独立し、あるワークスペースで作成したタグを別のワークスペースのタスクへ付与することはできない。ワークスペースを横断して全タスクを一覧する統合ビュー（例: 全ワークスペース共通の「今日」画面）は初期リリースのスコープ外とし、将来クライアント側機能として追加可能な拡張候補とする。

## 11. ローカルAI

### F-36 オンデバイス推論

AI機能はすべて端末上での推論により完結する。推論モデルはユーザーが任意にダウンロードするオプトイン方式とし、初期状態では無効とする。

### F-37 自然言語からのタスク作成

自然言語入力からタスクを自動生成する。例えば「明日15時に歯医者」という入力から、タイトルおよび作業予定日時（scheduled）を自動設定する。

### F-38 タスク分解提案

粒度の大きいタスクに対して、AIがサブタスク案を生成し提示する。提示された案をユーザーが取捨選択して採用する。

### F-39 週次レビューの要約生成【追加提案機能】

週次レビュー画面（F-23）の内容をもとに、AIが要約文を生成する。

## 12. 外部インターフェース

### F-40 CLI

コマンドラインインターフェースを提供し、タスクの一覧表示（list）・追加（add）・完了（done）・編集（edit）等の操作をサポートする。CLIはアプリ本体と同一のRustコアを介してローカルDBに直接アクセスする。

### F-41 MCPサーバー

ローカルのstdio上で動作するMCPサーバーを提供し、AIエージェントからタスク操作を可能にする。初回接続時にはユーザーの明示的な許可を必須とする。MCP経由で外部（AIエージェント側）にデータを渡すか否かはユーザー自身の選択として尊重する方針とし、無断でのデータ送出は行わない。

### F-42 エクスポート・インポート

タスクデータのエクスポート・インポートをJSON / Markdown / CSV形式でサポートする。加えて、暗号化されたバックアップファイル形式でのエクスポート・インポートをサポートする。

## 13. セキュリティ関連UX

### F-43 アプリロック

生体認証（Face ID / Touch ID / 指紋認証等）またはPINコードによるアプリロック機能を提供する。

### F-44 パスワード変更

ユーザーはパスワードを変更できる。パスワード変更に伴うデータの再暗号化が不要である理由については技術仕様書を参照する。

### F-45 スクリーンショット抑止（Android）

Android版において、アプリ画面のスクリーンショット・画面録画を抑止するオプションを提供する。

## 14. その他

### F-46 ホーム画面ウィジェット【追加提案機能】

iOS / Android のホーム画面ウィジェットとして、Todayタスク（F-12）の表示およびクイック追加操作を提供する。

### F-47 OS共有シートからのタスク作成【追加提案機能】

OS標準の共有シート（Share Sheet）経由で、他アプリから共有されたテキストやURLをもとにタスクを作成できる。

### F-48 多言語対応

顧客ターゲットはグローバルであり、初期リリースの時点から多言語対応を行う。基準言語（開発上のソース言語）は英語とし、初期リリースでは英語・日本語を提供する。以降の言語追加を迅速に行えるよう、以下のローカライズ基盤を最初から整備する。

- すべてのUI文字列を外部化し、ハードコードを禁止する。
- 複数形・性数の表現には ICU MessageFormat を用いて対応する。
- 翻訳管理プラットフォームを用いた継続的な翻訳ワークフローを構築する。

日付・時刻・数値・週の開始曜日などのロケール依存の表記についても、それぞれのロケールに応じた表示を行う。

RTL言語（アラビア語等）については、提供言語への追加自体は将来対応とするが、レイアウトは初期リリースの時点からRTL対応を前提に設計する（Flutter標準のRTLサポートを利用する）。

ストア掲載情報（App Store / Google Play のメタデータ）についても、提供言語分をローカライズする。

### F-49 アクセシビリティ

スクリーンリーダー対応、およびフォントサイズのスケーリングに対応する。

## 15. 非機能要件

### F-50 起動性能

コールドスタートは2秒以内に完了すること。

### F-51 描画性能

タスク数が1万件規模のリストにおいても、60fpsでの描画を維持すること。

### F-52 オフライン動作

同期機能（F-28, F-30）を除く全機能はオフラインで動作すること。

### F-53 クラッシュレポート

クラッシュレポートの送信はオプトインとし、送信データからは個人を特定しうる情報（PII）を除去すること。
