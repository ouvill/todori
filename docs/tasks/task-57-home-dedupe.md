# task-57: Home重複表示の解消とサブタスク親ラベル

## 1. 背景とコンテキスト

2026-07-08人間裁定で、task-55のHomeサブツリー同伴規則を改訂することになった。task-55では、子が親より早いHomeセクションに該当する場合、親配下にも早いセクションにも表示していた。この規則は、3階層それぞれに期日が付くケース（例: 孫=Overdue、子=Today、親=Tomorrow）で同一タスクが最大3回表示され、Homeのノイズになる。

以後Homeでは、各タスクを最大1回だけ表示する。自分の期日でOverdue / Today / Tomorrow / Upcomingに該当するタスクは、その該当セクションに単独行として表示する。期日なし等で自分ではHomeに該当しないタスクだけを、Homeに表示される直近の祖先の下に同伴する。同伴サブツリー構築時は、既に単独表示される子孫とその配下を剪定する。

また、単独表示された行がサブタスクである場合、リスト名ラベルは同一リスト内の冗長情報になりやすい。リスト名ラベルの代わりに、階層アイコン + 直近の親タスク名（1行省略）を表示し、semanticsにも親コンテキストを含める。

## 3. ゴール

- Home内で同一タスクが複数回表示されないようにする。
- 自分の期日でHomeセクションに該当するサブタスクは、該当セクションに単独行として表示する。
- 自分ではHomeに該当しない子孫は、Homeに表示される直近祖先の下へ同伴し続ける。
- 同伴サブツリー内では、単独表示される子孫とその配下を剪定する。
- 単独表示されたサブタスクには、リスト名ラベルの代わりに親タスク名ラベルを表示する。
- Homeの既存操作規則（チェック、スワイプ、詳細遷移、D&Dなし）を維持する。

### やること

1. Homeセクション構成ロジックを1タスク1表示へ変更する。
   - `isHomeTarget` かつ期日でHomeセクションに該当するタスクは、そのセクションの単独表示候補にする。
   - 親子関係にかかわらず、単独表示候補は自分の該当セクションへ置く。
   - 同一タスクが複数セクション、または親配下と単独行の両方に出ないようにする。
2. 同伴サブツリー構築時の剪定を実装する。
   - Homeに単独表示される子孫は、祖先の同伴サブツリーから外す。
   - 単独表示される子孫の配下も、その祖先側の同伴サブツリーから外す。
   - 期日なし等で自分では単独表示されない子孫は、Homeに表示される直近祖先の下に同伴する。
3. サブタスク単独行の文脈ラベルを実装する。
   - 親を持つ単独表示行では、リスト名ラベルを表示しない。
   - 代わりに、階層アイコン + 直近の親タスク名を1行省略で表示する。
   - 親タスク名ラベルはl10n/semanticsに対応し、スクリーンリーダーで親コンテキストが伝わるようにする。
   - 親を持たないルートタスクは従来どおりリスト名ラベルを表示する。
4. 既存操作規則との整合を確認する。
   - チェックトグル、スワイプ期日変更、行タップ詳細遷移は既存経路を維持する。
   - HomeではD&D並び替えを有効化しない。
   - セクション件数は、期日でそのセクションに該当する未完了Home対象タスク数を数え、同伴子孫を含めない既存方針を維持する。
5. widget testを追加・更新する。
   - 3階層×3期日（孫=Overdue、子=Today、親=Tomorrow等）のseedで各タスクが1回だけ表示されることを検証する。
   - 単独表示される子孫とその配下が祖先の同伴サブツリーから剪定されることを検証する。
   - 期日なし子は直近祖先の下に同伴され続けることを検証する。
   - サブタスク単独行の親ラベル、semantics、ルート行のリスト名ラベルを検証する。
6. visual QAを更新する。
   - 3階層×3期日seedを `home_tasks` に含める。
   - 目視で重複なし、剪定、親ラベル、ルート行のリスト名ラベルを確認できるスクリーンショットを生成する。

## 5. 実装手順（例）

1. `git status --short` で作業ツリーを確認する。
2. 2章のファイルを読み、現行の `_buildHomeSections`、`_hasSuppressingHomeAncestor`、`_buildHomeTaskRow`、`AppHomeTaskRow`、Home用fake seedを把握する。
3. Home対象タスクごとに、自分の期日から該当セクションを決める `targetSectionByTaskId` を作る。
4. 「単独表示されるタスクID集合」を先に確定し、セクションごとの単独表示rootを構成する。
5. 単独表示rootの同伴サブツリーを構築するとき、別の単独表示タスクIDに到達したら、そのノードと配下を追加せずに剪定する。
6. 期日なし子孫は、剪定対象でない限り、直近の単独表示祖先の下に残す。
7. 単独表示rootがサブタスクかどうかを判定し、親タスク名を `AppHomeTaskRow` に渡せる形へ整理する。
8. `AppHomeTaskRow` 側で、ルート行はリスト名ラベル、サブタスク単独行は階層アイコン + 親タスク名ラベルを表示する。必要な文言はARB化し、`flutter gen-l10n` を実行する。
9. widget testとvisual QA seedを更新する。
10. 品質ゲートを実行し、指示書末尾に `## 9. 完了報告` を追記する。

## 7. 制約・注意事項

- `docs/design/ui-spec.md` の2026-07-08人間裁定（Home重複表示の解消）を正とする。task-55の旧「両方に表示」規則へ戻さない。
- Homeセクション定義は変えない。期日ありタスクは既存のOverdue / Today / Tomorrow / Upcoming判定に従う。
- 期日なしルートタスクをHomeへ新規表示しない。
- 同伴サブツリーの剪定は表示上の剪定であり、タスクの親子関係やDB上のデータは変更しない。
- サブタスク単独行の親ラベルは直近の親のみを表示する。祖父母以上のパンくずは出さない。
- 親タスク名が長い場合は1行省略し、行全体の高さや右側メタデータを不自然に押し出さない。
- UI文字列を追加する場合はARB化する。`app/tool/check_hardcoded_strings.sh` に検出される直書きを追加しない。
- 既存の階層ガイド、チェックボックス幾何、Home横幅圧縮、48px級タップ領域を崩さない。
- Rust APIを変更する場合は、変更理由を完了報告に記録し、FRB生成物をcodegenで更新する。ただし本タスクは原則としてFlutter側表示ロジックの変更で完結させる。
